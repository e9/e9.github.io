ActiveRecordでattr_accessorとselectを併用すると

 2016-05-19 No tag 
$ bin/rails -v
Rails 5.0.0.rc1

こういうモデルがあるとする。

class Apple < ApplicationRecord
  attr_accessor :y
end

実装内容の良し悪しはさておき、下記を実行する。

irb(main):001:0> apple = Apple.select("1 as x", "2 as y").first
irb(main):002:0> apple.x
=> 1

ではapple.yには何が入っているか。

irb(main):003:0> apple.y
=> nil
irb(main):004:0> apple.as_json
=> {"id"=>nil, "x"=>1, "y"=>nil}

nilなのだ。attr_accessorの#yが優先されるらしい。SELECTした結果にアクセスするには[:y]とすればよい。

irb(main):005:0> apple[:y]
=> 2

attr_accessorをひとつ置いただけで、既存のコードが動かなくなる可能性をはらんでいる。というか、それでさっきハマったのだ。

このあとxとyにそれぞれ代入すると、さらにわけがわからない。

irb(main):006:0> apple.x = 3
irb(main):007:0> apple.y = 4
irb(main):008:0> apple.as_json
=> {"id"=>nil, "x"=>3, "y"=>4}
irb(main):009:0> apple.x
=> 3
irb(main):010:0> apple[:x]
=> 3
irb(main):011:0> apple.y
=> 4
irb(main):012:0> apple[:y]
=> 1

動的言語は強力だが、ときに怖い。
